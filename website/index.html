What It Does

Capabilities

Decompose complex tasks, distribute across heterogeneous workers, and synthesize unified deliverables.

  1. 01 Decompose

    Break your task into 2-5 independent subtasks. Each worker gets a self-contained prompt with all context it needs.

  2. 02 Spawn

    Launch OpenCode and Codex workers as actual OS processes with isolated workdirs. No subagents — real CLI processes.

  3. 03 Monitor

    Batch-style monitoring via subprocess output capture. Inspect logs before trusting files alone.

  4. 04 Evaluate

    Score every worker against a rubric: Presence, Correctness, Completeness, Quality. Pass threshold: average >= 3.0.

  5. 05 Respawn

    Failed workers get specific corrective feedback and are respawned. Max 2 respawns per subtask. Circuit breaker detects rate limits.

  6. 06 Rebalance

    Shift work between OpenCode and Codex pools based on capacity, reliability, and task fit. Not a one-way fallback chain.

  7. 07 Dynamic Spawn

    After the first batch, spawn additional workers if gaps remain or user feedback arrives mid-orchestration.

  8. 08 Synthesize

    Merge all worker outputs into a single coherent deliverable, resolve conflicts, and credit which worker produced what.

System Design

Architecture

Hermes acts as conductor. Workers are the orchestra. The manifest is the score.

Hermes (Orchestrator)
    │
    ├── Discover → .worker-registry.json
    ├── Decompose → manifest.json
    ├── Spawn → adapter_cli | adapter_http | adapter_python | adapter_docker
    ├── Monitor → unified monitor_result schema
    ├── Evaluate → rubric-based scoring
    ├── Rebalance → respawn / phase-gate
    ├── Track → cost + tokens ledger
    └── Synthesize → unified deliverable

Get Started

Quickstart

Pure stdlib bootstrap. No dependencies required for core orchestration.

# 1. Clone the skill
git clone https://gitlab.com/hackafrik/worker-orchestrator.git
cd worker-orchestrator

# 2. Discover available workers
python3 scripts/discover_workers.py

# 3. Write a manifest
cat > manifest.json << 'EOF'
{
  "phases": [
    {
      "phase_number": 0,
      "tasks": [
        {
          "task_id": "demo-worker-1",
          "worker_category": "PYTHON_SCRIPT",
          "adapter_name": "python3",
          "prompt": "Write a CSV reader",
          "workdir": "/tmp/demo",
          "timeout_seconds": 300,
          "rubric": {
            "expected_files": ["csv_reader.py"]
          },
          "runtime": {
            "script_path": "scripts/demo.py",
            "interpreter": "python3"
          }
        }
      ]
    }
  ]
}
EOF

# 4. Run the orchestrator
python3 scripts/run_orchestrator.py manifest.json
